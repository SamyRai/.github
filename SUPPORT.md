# Getting Support from Glowing Pixels UG

Need help with a GLPX repository or service? Here is how to get it!

## Internal Documentation
Before opening an issue, please check the internal documentation:
- **Global Docs Hub:** `https://docs.glpx.pro`
- **Architecture & Topology:** Check the `ARCHITECTURE.md` in the `cluster` repository.

## Opening an Issue
If you've encountered a bug or have a feature request, please open an issue in the relevant repository on our internal Gitea instance (`https://gitea.bk.glpx.pro`).
Please use the provided Issue Templates (`Bug Report` or `Feature Request`) to ensure you provide the necessary context (like whether it affects the `Baikonur` or `Mir` cluster).

## Security Vulnerabilities
If you discover a security vulnerability, **do not open a public issue**.
Please refer to our [Security Policy](SECURITY.md) and email `security@glpx.pro`.

## Direct Contact
For urgent infrastructure incidents, reach out to `@mukimovd` or ping the `#devops` channel.
